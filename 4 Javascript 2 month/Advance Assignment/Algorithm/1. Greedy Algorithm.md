# 📌 Greedy Algorithm – Complete Notes (DSA)

## 🔷 What is Greedy Algorithm?

Greedy Algorithm ek aisi approach hai jisme hum **har step par locally best (optimal) choice** lete hain, with the hope ki final result globally optimal hoga.

👉 Simple line (Interview):

> “Greedy algorithm makes the best choice at each step without considering future consequences.”

---

## ⚙️ Key Characteristics

* ✅ Local optimum choose karta hai
* ✅ Future reconsider nahi karta
* ✅ Fast and efficient (mostly O(n) or O(n log n))
* ❌ Har problem me kaam nahi karta (DP vs Greedy confusion)

---

## 🧠 Greedy kab kaam karta hai?

Greedy tab work karta hai jab:

1. **Greedy Choice Property**
   → Local best choice leads to global best

2. **Optimal Substructure**
   → Problem ko chhote parts me tod sakte hain

---

## 🔁 General Greedy Algorithm Steps

```text
1. Problem ko samjho
2. Greedy choice define karo (best local decision)
3. Data ko sort karo (agar required ho)
4. Loop chalao aur decision lo
5. Final result return karo
```

## 💻 Example (Roman to Integer – Greedy)

```javascript
const romanToInt = function(s) {
    const map = {
        'I': 1, 'V': 5, 'X': 10,
        'L': 50, 'C': 100,
        'D': 500, 'M': 1000
    };

    let result = 0;

    for (let i = 0; i < s.length; i++) {
        if (i + 1 < s.length && map[s[i]] < map[s[i + 1]]) {
            result -= map[s[i]];
        } else {
            result += map[s[i]];
        }
    }

    return result;
};
```

## ⏱️ Complexity

* Time: **O(n)** / **O(n log n)** (depends on sorting)
* Space: **O(1)**

## ⚠️ Greedy vs DP

| Greedy             | DP               |
| ------------------ | ---------------- |
| Fast               | Slower           |
| Local decision     | Global decision  |
| No backtracking    | Uses memoization |
| Not always correct | Always correct   |

---

## 🎯 Interview Tips

* Always check: “Kya local decision global optimum de raha hai?”
* Sorting + single pass = strong greedy hint
* Agar doubt ho → DP try karo
